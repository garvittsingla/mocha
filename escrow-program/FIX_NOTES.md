# Debugging `anchor build` ‚Äî `constant_time_eq` edition2024 Error

## The Error

```
error: failed to download `constant_time_eq v0.4.2`
Caused by: feature `edition2024` is required
  The package requires the Cargo feature called `edition2024`, but that feature is
  not stabilized in this version of Cargo (1.84.0 ...)
```

---

## Investigation

### Step 1 ‚Äî Identify which Cargo was failing

The host machine had Rust `1.89.0` active (via `rust-toolchain.toml`), yet the error
reported **Cargo 1.84.0**. That was the first clue the failing Cargo wasn't the system one.

Running:

```bash
ls ~/.cache/solana/
# v1.51   v2.3.3
~/.cache/solana/v1.51/platform-tools/rust/bin/cargo --version
# cargo 1.84.0
```

Confirmed: `anchor build` uses its own **bundled Solana BPF platform-tools** (in
`~/.cache/solana/v1.51/`), not the system Rust. That bundled Cargo is `1.84.0`, which
predates the stabilization of `edition2024` (stabilized in `1.85.0`).

### Step 2 ‚Äî Trace the dependency chain

```bash
cargo update -p constant_time_eq --precise 0.3.1
# error: failed to select a version for `constant_time_eq = "^0.4.2"` required by blake3 v1.8.3
#   ... blake3 required by solana-program v2.3.0
#   ... solana-program required by spl-associated-token-account v7.0.0
#   ... required by anchor-spl v0.32.1
```

Full chain:

```
escrow-program
  ‚îî‚îÄ anchor-spl 0.32.1
       ‚îî‚îÄ spl-associated-token-account 7.0.0
            ‚îî‚îÄ solana-program 2.3.0
                 ‚îî‚îÄ blake3 1.8.3          ‚Üê introduced constant_time_eq 0.4.2
                      ‚îî‚îÄ constant_time_eq 0.4.2  ‚Üê requires edition2024 ‚ùå
```

`blake3 1.8.3` pinned `constant_time_eq` to `^0.4.2`, so the only way to downgrade
`constant_time_eq` was to also downgrade `blake3`.

### Step 3 ‚Äî Verify a compatible older `blake3` exists

`blake3 1.5.5` uses `constant_time_eq ^0.3.x`, which has no `edition2024` requirement
and compiles cleanly on Cargo 1.84.0.

### Step 4 ‚Äî Why we couldn't use `[patch.crates-io]`

An attempted `[patch.crates-io]` override was rejected by Cargo:

```
error: patch for `constant_time_eq` points to the same source,
but patches must point to different sources
```

Cargo only allows `[patch]` to redirect to a _different_ source (e.g., a git repo or
local path), not to a different version on the same registry.

---

## Fix Applied

A single `cargo update` command downgraded two crates in `Cargo.lock` simultaneously:

```bash
cargo update -p blake3 --precise 1.5.5
#  Downgrading blake3 v1.8.3 -> v1.5.5
#  Downgrading constant_time_eq v0.4.2 -> v0.3.1
```

No changes to `Cargo.toml` were needed ‚Äî only `Cargo.lock` was updated.

```
blake3 1.5.5  ‚Üí  constant_time_eq 0.3.1  ‚úÖ  (no edition2024 required)
```
